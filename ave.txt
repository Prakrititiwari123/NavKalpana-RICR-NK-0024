================================================================================
                      KEY FEATURES & WORKFLOWS
================================================================================

1. USER AUTHENTICATION WORKFLOW:
   Frontend (Login.jsx) → authService.js → Backend (authRouter.js) 
   → authController.js → userModel.js → Database
   → JWT token generation (authToken.js) → AuthContext storage

2. FITNESS TRACKING WORKFLOW:
   Dashboard.jsx → workoutService.js → Backend (routes)
   → Controllers → Database → reportsService.js → Charts & Analytics

3. AI-POWERED FEATURES:
   Chat.jsx → Backend (aiRouter.js) → aiController.js → Groq AI API
   → AI-generated recommendations → Frontend display

4. DIET MANAGEMENT:
   DietPlan.jsx/CalorieTracker.jsx → Backend APIs
   → MacroCalculator calculations → Database storage

5. REPORT GENERATION:
   ReportCharts.jsx → reportsService.js → Backend data aggregation
   → PDF generation or Email sending (emailService.js)

6. PROFILE MANAGEMENT:
   ProfileSetup.jsx → profileService.js → Backend (userController.js)
   → imageUploader.js (if profile picture upload) → Cloudinary

================================================================================
                       DEPLOYMENT & RUNNING
================================================================================

BACKEND START:
  • Command: npm run dev (development) or npm start (production)
  • Location: backend/
  • Port: Usually 5000 or 8000 (check server.js)
  • Requires: Node.js, MongoDB, .env file with API keys

FRONTEND START:
  • Command: npm run dev (development)
  • Location: frontend/
  • Port: Usually 5173 (Vite default)
  • Requires: Node.js, environment variables (API URL)

BUILD FOR PRODUCTION:
  • Frontend: npm run build (creates dist/ folder)
  • Backend: Typically runs from source or containerized

================================================================================
                         DEVELOPMENT NOTES
================================================================================

1. Environment Variables needed:
   Backend: MongoDB URI, JWT Secret, Cloudinary keys, Groq API key, Email config
   Frontend: VITE_API_BASE_URL (backend API endpoint)

2. Authentication Flow:
   - Uses JWT tokens stored in context/local storage
   - Protected routes via authMiddleware in backend
   - PrivateLayout wrapper for frontend routes

3. State Management:
   - React Context (AuthContext) for auth state
   - Component state for local UI state
   - Backend session via JWT

4. Error Handling:
   - API response interceptors in config/Api.jsx
   - Try-catch blocks in services
   - Error messages displayed in components
